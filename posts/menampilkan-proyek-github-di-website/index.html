Cara Menampilkan Proyek GitHub di Website Pribadi

Pelajari bagaimana saya mengotomatisasi penampilan proyek GitHub di website Hugo saya menggunakan GitHub API

Sebagai seorang developer, saya ingin menampilkan proyek-proyek GitHub saya di website pribadi tanpa harus memperbaruinya setiap kali membuat atau memperbarui repository. Dalam artikel ini, saya akan menjelaskan bagaimana saya mengotomatisasi proses ini menggunakan GitHub API dan Hugo.

Tantangan#

Saya ingin website saya secara otomatis menampilkan proyek-proyek GitHub saya, termasuk:

  • Nama dan deskripsi proyek
  • Bahasa utama dan topik
  • Jumlah bintang dan fork
  • Tautan ke repository

Tantangannya adalah mengambil data ini secara otomatis dan menampilkannya dengan rapi di website saya.

Solusi#

1. Mengambil Data dari GitHub API#

Saya membuat script bash (fetch-github-data.sh) yang menggunakan GitHub API untuk mengambil informasi tentang repository saya:

#!/usr/bin/env bash

# Fungsi untuk mengambil repository GitHub dengan topik
fetch_github_repos() {
  local username="pararang"  # Ganti dengan username GitHub Anda
  local output_file="data/projects.json"
  
  echo "Mengambil repository GitHub untuk $username..."
  
  # Buat direktori data jika belum ada
  mkdir -p "$(dirname "$output_file")"
  
  # Periksa apakah jq terinstal
  if ! command -v jq &> /dev/null; then
    echo "Error: jq tidak terinstal. Silakan install jq untuk memproses data JSON."
    echo "Di macOS: brew install jq"
    echo "Di Ubuntu/Debian: sudo apt-get install jq"
    exit 1
  fi
  
  # Ambil repository menggunakan GitHub API
  # Ini mengambil repository publik yang diurutkan berdasarkan terakhir diperbarui
  # Filter repository fork dan yang tidak memiliki deskripsi
  repos_response=$(curl -s "https://api.github.com/users/$username/repos?sort=updated&direction=desc&per_page=100")
  
  # Periksa apakah kita mengenai batas rate
  if echo "$repos_response" | jq -e '.message' | grep -q "API rate limit exceeded"; then
    echo "Peringatan: Batas rate API terlampaui. Menggunakan file data yang ada jika tersedia."
    if [ -f "$output_file" ]; then
      echo "Menggunakan file data yang ada."
      return 0
    else
      echo "Error: Tidak ada file data yang ditemukan dan batas rate API terlampaui."
      exit 1
    fi
  fi
  
  repos=$(echo "$repos_response" | jq -c 'map(select(.fork == false and .description != null) | {name, description, url: .html_url, stars: .stargazers_count, forks: .forks_count, language, updated_at})')
  
  if [ -z "$repos" ] || [ "$repos" = "null" ]; then
    echo "Error: Gagal mengambil repository GitHub"
    exit 1
  fi
  
  # Buat file sementara untuk menyimpan data gabungan
  temp_file=$(mktemp)
  
  # Proses setiap repository untuk menambahkan topik
  echo "[" > "$temp_file"
  first=true
  
  echo "$repos" | jq -c '.[]' | while read -r repo; do
    # Dapatkan nama repository
    name=$(echo "$repo" | jq -r '.name')
    
    # Gabungkan bahasa dan topik menjadi techStack
    # Tangani bahasa null dengan mengubahnya menjadi array kosong
    if [ "$(echo "$repo" | jq -r '.language')" = "null" ]; then
      lang_array="[]"
    else
      lang_array="[$(echo "$repo" | jq -r '.language' | jq -R .)]"
    fi
    
    # Ambil topik untuk repository ini
    echo "Mengambil topik untuk $name..."
    
    topics_response=$(curl -s "https://api.github.com/repos/$username/$name/topics" -H "Accept: application/vnd.github.mercy-preview+json")
    
    # Periksa apakah kita mengenai batas rate
    if echo "$topics_response" | jq -e '.message' | grep -q "API rate limit exceeded"; then
      echo "Peringatan: Batas rate API terlampaui. Menggunakan array kosong untuk topik."
      topics="[]"
    else
      topics=$(echo "$topics_response" | jq -c '.names // []')
    fi
    
    # Gabungkan bahasa dan topik menjadi techStack
    tech_stack=$(echo "{\"lang\": $lang_array, \"topics\": $topics}" | jq -c '.lang + .topics | unique')
    
    # Tambahkan techStack ke data repository
    if [ "$first" = true ]; then
      first=false
    else
      echo "," >> "$temp_file"
    fi
    
    echo "$repo" | jq --argjson techStack "$tech_stack" '. + {techStack: $techStack}' >> "$temp_file"
  done
  
  echo "]" >> "$temp_file"
  
  # Format output akhir
  jq '.' "$temp_file" > "$output_file"
  
  # Bersihkan
  rm "$temp_file"
  
  echo "Repository GitHub (tanpa fork dan tanpa deskripsi) dengan tech stack disimpan di $output_file"
}

# Jalankan fungsi
fetch_github_repos

3. Menampilkan Proyek di Website#

Saya membuat layout khusus (layouts/projects/single.html) untuk menampilkan proyek-proyek:

{{ define "main" }}
<article class="post-single">
  <header class="post-header">
    <h1 class="post-title">{{ .Title }}</h1>
  </header>
  <div class="post-content">
    {{ .Content }}
    
    {{ if site.Data.projects }}
    <div class="projects-list">
      {{ range site.Data.projects }}
      <div class="project-item">
        <h2><a href="{{ .url }}" target="_blank">{{ .name }}</a></h2>
        <p>{{ .description }}</p>
        <div class="project-meta">
          {{ if .techStack }}
          {{ range .techStack }}
          <span class="project-language">{{ . }}</span>
          {{ end }}

4. Menampilkan Proyek Unggulan di Halaman Depan#

Saya juga mengonfigurasi proyek-proyek tertentu untuk ditampilkan di halaman depan dengan menambahkannya ke konfigurasi Hugo (hugo.yaml):

params:
  homeInfoParams:
    PinnedProjects:
      - mengcp
      - vue-covid
      - nams-thesis-fuzzy-delphi

Dan memperbarui template halaman depan (layouts/partials/home_info.html) untuk menampilkan proyek-proyek yang disematkan:

{{- if site.Data.projects }}
<div class="pinned-item">
    <h2>Proyek Unggulan</h2>
    {{- range site.Params.homeInfoParams.PinnedProjects }}
    {{- $projectName := . }}
    {{- $project := (where site.Data.projects "name" $projectName) }}
    {{- if $project }}
    {{- range $project }}
    <div class="project-item">
        <h3><a href="{{ .url }}" target="_blank">{{ .name }}</a></h3>
        <p>{{ .description }}</p>
        {{- with .techStack }}
        <div class="tech-stack">
            {{- range . }}
            <span class="tech-tag">{{ . }}</span>
            {{- end }}
        </div>
        {{- end }}
    </div>
    {{- end }}
    {{- end }}
    {{- end }}
</div>
{{- end }}

Manfaat dari Pendekatan Ini#

  1. Pembaruan Otomatis: Halaman proyek saya selalu terbaru dengan repository GitHub saya
  2. Informasi Kaya: Pengunjung dapat melihat tidak hanya nama proyek, tetapi juga bahasa, topik, bintang, dan fork
  3. Kontrol Manual: Saya masih dapat mengontrol proyek mana yang ditampilkan di halaman depan
  4. Manajemen Batas Rate: Saya menjalankan script pengambilan data secara manual untuk menghindari batas rate API
  5. Tidak Ada Dependensi Eksternal: Semua ditangani dalam ekosistem Hugo

Kesimpulan#

Solusi ini memungkinkan saya untuk menampilkan proyek GitHub saya secara otomatis namun terkendali. Dengan menjalankan script pengambilan data secara manual ketika saya ingin memperbarui data proyek saya, saya menghindari masalah batas rate sambil tetap menyediakan informasi yang segar dan relevan kepada pengunjung website saya.

Kombinasi data GitHub API, template data Hugo, dan layout khusus menciptakan pengalaman yang mulus untuk menampilkan pekerjaan saya tanpa pemeliharaan manual.
